# 这套考研记录系统怎么用

这不是一个有界面的软件，它的逻辑是：

1. 在 `data` 里的 Markdown 文件记录学习过程。
2. 用 `py` 里的 Python 脚本把记录转换成 Excel。
3. 再用统计脚本生成处理后的 Excel 和图片。
4. 最后看 `data/xlsx` 和 `data/pic` 里的结果复盘。

## 每天主要改什么

最常用的是这三个文件：

- `data/学习时长.md`：记录今天每个科目学了多久。
- `data/正确率.md`：阶段性记录一本书、一章题、英语阅读等正确率。
- `data/学习状态.md`：记录当天计划、完成情况、状态评分和复盘。

`data/规划.md` 是长期计划，不需要天天改，适合每周或每月调整一次。

## 学习时长怎么记

在 `data/学习时长.md` 里找对应的表格，比如数学、英语、cs。

格式照着原来的写：

```md
| 6.29 | 8:30-11:30, 14:00-16:00, 19:00-20:30 | 做完高数第3章 |
```

意思是：

- 第一列：日期。
- 第二列：学习时间段，可以写多个，用英文逗号隔开。
- 第三列：备注，可以写今天完成了什么。

如果某个备注代表一个阶段完成，脚本会把从上一个阶段到这里的总时长算出来。

## 正确率怎么记

在 `data/正确率.md` 里找对应科目，比如高数、线代、概率论。

格式照着原来的表格写：

```md
| 660题 | 一刷 | 6.1-6.10 | 337/380=88.7% | ... |
```

这里最重要的是 `做对题数/总题数=正确率%` 这个格式。脚本会检查比例是否对得上，并生成新的统计表和图。

英语阅读比较特殊，它在 `acc-table-阅读` 里用一串数字记录。比如每篇阅读 5 道题，数字 `4` 就表示那篇对了 4 道。

## 状态怎么记

在 `data/学习状态.md` 里每天加一行：

```md
| 6.29 | 今天计划 | 4 | 3 | 3 | 4 | 5h20 | 2h10 | 今天复盘 |
```

它的作用不是为了精确统计，而是帮你发现：

- 是不是计划总是定太满。
- 是不是状态差时正确率明显下降。
- 手机/电脑时长是不是影响学习。
- 哪些阶段是真学懂了，哪些只是赶进度。

## 一键更新

进入 `py` 文件夹：

```bash
cd ~/Downloads/kaoyan-main/py
```

然后运行：

```bash
python3 auto_run.py
```

我已经把 `auto_run.py` 调整成适合 Mac 的版本，并把绘图需要的库放在项目自己的 `.python-libs` 文件夹里。以后优先用 `auto_run.py` 一键跑。

如果只是把 Markdown 转成 Excel，可以只运行：

```bash
python3 md2xlsx.py
```

## 输出看哪里

- 原始转换表格：`data/xlsx/学习时长-xxx.xlsx`、`data/xlsx/正确率-xxx.xlsx`
- 处理后表格：`data/xlsx/P-学习时长-xxx.xlsx`、`data/xlsx/P-正确率-xxx.xlsx`
- 图片结果：`data/pic`

## 推荐你自己的使用节奏

每天晚上做三件事：

1. 在 `学习时长.md` 加今天的时间。
2. 在 `学习状态.md` 加今天的状态和一句复盘。
3. 如果今天做完一章题或一本书的一部分，在 `正确率.md` 加正确率。

每周做一次：

1. 运行 `python3 auto_run.py`。
2. 看 `data/xlsx` 和 `data/pic`。
3. 对照 `规划.md` 调整下周计划。

这套系统最有价值的地方不是表格漂亮，而是它把“我今天学了多久、做题质量怎么样、状态为什么波动”串起来了。你照这个节奏用，后面复盘会很清楚。
