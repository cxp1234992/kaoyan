# 我的考研学习系统

这套模板保留原仓库最有用的逻辑：计划、时长、正确率、状态、复盘。

我把它改成更适合你自己从零开始用的版本。不要先追求复杂，先每天写三件事：

1. 今天学了什么。
2. 学了多久。
3. 哪些地方错了、卡了、下次怎么改。

## 每天怎么用

每天晚上打开当天的每日记录文件，补几行就够：

- `每日记录/YYYY-MM-DD.md`

如果当天没有做成套题，正确率表可以留空。

## 本地写，GitHub 看图

本地写完后，运行：

```bash
python3 tools/generate_personal_dashboard.py
```

它会自动生成：

- `可视化/README.md`
- `可视化/每日学习时长.png`
- `可视化/各科学习总时长.png`
- `可视化/正确率趋势.png`
- `可视化/学习时长汇总.csv`
- `可视化/正确率汇总.csv`

推到 GitHub 后，打开 `我的考研系统/可视化/README.md` 就能看到图。

如果你想一键生成、提交、推到 GitHub，可以在仓库根目录运行：

```bash
bash update_my_study.sh
```

网络不好时，脚本最后一步可能推不上去；本地记录和图不会丢，等网络好再运行一次即可。

## 每周怎么用

每周日打开：

- `05-周复盘.md`

回答三个问题：

1. 哪科进步了？
2. 哪科只是花了时间但没效果？
3. 下周要砍掉什么、加强什么？

## 每月怎么用

每月底打开：

- `06-月复盘.md`

重点看趋势，不要只看某一天状态。

## 文件说明

| 文件 | 用途 |
| --- | --- |
| `00-总目标.md` | 目标院校、分数、阶段安排 |
| `模板-每日记录.md` | 每天复制一份用 |
| `每日记录/` | 每天一个文件 |
| `可视化/` | 自动生成的图和汇总 |
| `01-每日记录.md` | 旧版总表，可不用 |
| `02-学习时长.md` | 旧版总表，可不用 |
| `03-正确率.md` | 旧版总表，可不用 |
| `04-错题复盘.md` | 记录真正值得回看的错题 |
| `05-周复盘.md` | 每周调整节奏 |
| `06-月复盘.md` | 每月看大方向 |
| `07-资料清单.md` | 管理书、课、题、软件 |

## 这套系统的原则

- 记录要轻，不要把记录本身变成负担。
- 时间只能说明投入，正确率和复盘才说明质量。
- 每周必须调整计划，不要机械执行已经不适合自己的安排。
- 错题不是越多越好，只收录能暴露薄弱点的题。
- 状态差不等于今天废了，但状态连续差三天就必须改节奏。
